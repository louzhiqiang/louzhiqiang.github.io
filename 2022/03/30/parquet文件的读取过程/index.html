parquet文件的读取过程

最近在修改hudi读取复杂数据的bug，然后钻研了一下parquet文件的读取，这里记录一下。

hudi本身实现的是向量化读取，而parquet-tool实现的是基础算法。这里先研究一下这个基础算法，最后再来看怎么修改那个向量化读取的算法。

关于parquet的基础概念，之前有记录过，可以翻看一下历史记录。

ParquetReader的创建

创建过程会有一些初始化，这里是parquet文件的入口，也是算法入口，可以知道整个过程用到了哪些关键元素。

reader读取

reader初始化完了，现在看看如何进行数据的读取。

读取主要调用的是read()方法

初始调用的时候，会去初始化reader。

InternalParquetRecordReader 是很关键的类，内部会初始化 ColumnIOFactory 。ColumnIOFactory 这个类稍后会着重再介绍一遍。

fileSchema 和 requestedSchema 几乎一样的属性信息，都是 MessageType ，columnCount 是指实际包含的列数，也就是整个 元数据打平之后的信息，因为实际上Path就是每一个路径，代表的是根节点到叶子节点的所有路径数量，也就代表了有多少列。

在这里调用了 prepareForRead 这个方法，这里会初始化 RecordMaterializer ，这个是物化的关进类。要关注一下他在哪里被调用这件事。

SimpleRecordMaterializer对象生成的时候，生成了 SimpleRecordConverter ，这个就是按照类型转化数据的对象。

Top-level class which should be implemented in order to materialize objects from a stream of Parquet data. Each record will be wrapped by GroupConverter.start() and GroupConverter.end(), between which the appropriate fields will be materialized.

这是RecordMaterializer 这个对象的注释。

根据数量和字段类型创建 converters。这个时候创建的 converter的数量是一级字段的数量。也就是未展开之前的。

在这里如果是原生数据类型就直接创建一个SimplePrimitiveConverter，反之就创建一个SimpleRecordConverter，也就是记录级别的。

非原生类型的都会走 SimpleRecordConverter ，生成一个完整对象的converter。直到遇到所有的数据类型都是原生数据类型。

这里稍微捋一下 这个converter的生成最终形成的结构就是 整个schema树形结构的一个映射，然后每个叶子节点就是最终的coverter的对象。

到这里 initReader 的逻辑就完事儿了。主要业务逻辑就是生成一个 InternalParquetRecordReader 对象。

下边就开始读取数据记录了。

record记录数据

读取数据记录主要是方法 nextKeyValue。在对象 InternalParquetRecordReader 中。

checkRead的主要逻辑如下：

1、PageReadStore pages = reader.readNextRowGroup(); 获取rowGroup

2、MessageColumnIO 创建。主要就是创建每个叶子节点的 PrimitiveColumnIO 。

3、创建recordReader。逻辑比较复杂。这个方法里的应该也是这个算法的核心。这块稍后来剖析，看代码细节实在是不敢恭维。

1、recordRootConverter的start() 和 end()代表一条记录的开始和结束。在start里会创建一个 SimpleRecord 的对象。

2、逐个字段去取值。

3、这里state是一个状态机，依赖state来进行column的跳转。state是每一个叶子节点算 一个state。

4、recordMaterializer进行一行数据的物化。

整个这个算法实际上是 parquet的结构的一个完全的 映射式读取。如果要弄明白parquet这个文件的设计，实际上这个工具是一个很好的入口。

关于逐层数据读取的过程，实际上是依赖于 RecordReaderImplementation 这个类的初始化以及read()方法。这俩是最核心的点。

当前的读取方式并非是向量化的方式，但是主体流程是值得参考的，后边针对spark如何向量化读取parquet的需要重新看一遍流程，可以对比学习。
